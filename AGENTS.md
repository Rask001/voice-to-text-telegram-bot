# AGENTS

Before working on this project, use the project documentation as the map.

1. Read `docs/AGENTS.md`.
2. Read `docs/PROJECT_MAP.md`.
3. If the task affects flows or architecture, read `docs/ARCHITECTURE.md`.
4. If the task is about a feature, read `docs/FEATURES.md`.
5. Open only the specific source files needed after that.
6. Update docs after changes to architecture, env, handlers, database, callback data, deployment, or public bot behavior.

Do not scan the whole repository first unless the docs are missing, stale, or insufficient for the task.
